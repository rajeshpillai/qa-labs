KATA 24

API-First Testing - Testing APIs Without a Browser UI

What is API-First Testing?

Testing the API Layer Directly

In API-first testing, you send HTTP requests directly to an API endpoint and verify the response — without loading any browser UI. This is faster than UI tests, catches backend bugs earlier, and is essential for validating API contracts.

Playwright provides request context for direct HTTP calls. Cypress provides cy.request() for the same purpose. Both bypass the browser's rendering engine entirely.

When to Use API-First Testing

  • Validating API response shapes and status codes
  • Testing CRUD operations (Create, Read, Update, Delete)
  • Verifying authentication and authorization rules
  • Testing error handling and edge cases
  • Setting up test data before UI tests (seeding)
  • Performance baseline testing (response times)

API Endpoints Reference

The exercises in this kata use https://jsonplaceholder.typicode.com, a free public REST API for testing. No authentication required.

GET /posts Returns a list of 100 posts. Each post has: id, userId, title, body.
GET /posts/:id Returns a single post by ID. Returns 404 for non-existent IDs.
POST /posts Creates a new post. Send JSON body with { title, body, userId }. Returns the created post with an id.
GET /posts/:id/comments Returns all comments for a specific post. Each comment has: id, postId, name, email, body.
GET /users Returns a list of 10 users. Each user has: id, name, username, email, address, phone, website, company.

Testing Tools Comparison

Feature Playwright Cypress
Send GET request request.get(url) cy.request('GET', url)
Send POST request request.post(url, { data }) cy.request('POST', url, body)
Set base URL test.use({ baseURL }) Cypress.config('baseUrl')
Custom headers request.get(url, { headers }) cy.request({ headers })
Access response body await response.json() .its('body')
Check status code expect(response.status()).toBe(200) .its('status').should('eq', 200)
