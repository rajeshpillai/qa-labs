KATA 37

Page Object Model (POM) Pattern

About This Kata

This kata does not introduce a new playground page. Instead, it teaches you the Page Object Model design pattern by writing POM classes/objects for the KYC Onboarding Form from Kata 26.

The POM pattern wraps page interactions into reusable classes (Playwright) or objects (Cypress). This keeps your tests clean and your selectors in one place.

Open Kata 26: KYC Onboarding Form

Without POM (Fragile)

// Every test repeats selectors — if one changes, ALL tests break
await page.getByTestId('full-name-input').fill('Aisha Patel');
await page.getByTestId('email-input').fill('aisha@example.com');
await page.getByTestId('country-select').selectOption('IN');
await page.getByTestId('submit-btn').click();

With POM (Robust)

// Selectors live in ONE file — tests read like user stories
const kycForm = new KycFormPage(page);
await kycForm.fillName('Aisha Patel');
await kycForm.fillEmail('aisha@example.com');
await kycForm.selectCountry('IN');
await kycForm.submit();
